# Getting Started

도울정보기술 개발문서 서비스입니다. 본 서비스에서는 장고/리액트/DevOps와 관련된 내용을 담고 있습니다.

## 장고

파이썬으로 개발된 웹프레임워크이며, 중복을 최대한 줄이고 비즈니스 로직에 집중할 수 있도록 도와줍니다. 장고와 함께 `django-rest-framework` 라이브러리가 함께 사용됩니다.

## 리액트

보다 손쉬운 웹프론트엔드 UI 개발을 위한 라이브러리입니다. 기존의 웹페이지와도 연동이 가능하며, 대개 SPA (Single Page Application) 목적으로 활용이 많이 됩니다. 간단한 웹페이지는 jQuery 만으로 충분할 수 있지만, 리액트를 활용한다면 보다 체계적으로 "웹 컴포넌트" 기반의 개발을 할 수 있으며, 이를 통해 개발 생산성을 극대화시킬 수 있습니다.

## 컨테이너 기술

소프트웨어를 팩키징하는 하나의 방법입니다. 웹 애플리케이션 뿐만 아니라, 거의 모든 애플리케이션을 컨테이너로 팩키징할 수 있습니다. 팩키죙된 컨테이너를 이미지라고 부릅니다. 이 이미지와 컨테이너 실행환경만 있다면, 어떤 OS/클라우드 환경에서도 손쉽게 즉시 컨테이너를 구동할 수 있습니다. 물류에서 컨테이너 형태로 손쉽게 육로/해로로 물류를 수송할 수 있는 것과 같은 이치입니다.

컨테이너 기술 중에 가장 활발히 사용되는 것이 `Docker` 기술입니다.

## Docz

리액트 기반의 정적 개발문서 생성 라이브러리입니다. 프로젝트를 빌드하여 정적 HTML 파일들을 생성해냅니다. 빌드된 결과물은 HTML이기에 검색엔진최적화 이슈도 해결됩니다. 범용적인 정적 사이트 생성기인 [Gatsby](https://www.gatsbyjs.org/)로 개발되었습니다. Gatsby의 경우 블로그나 학교소개 등 컨텐츠 업데이트 주기가 긴 서비스의 경우 활용하시면 좋습니다.

프로젝트가 nodejs/react로 구성되기에 빌드머신에 nodejs/react를 설치하여 빌드를 진행할 수도 있지만, Docker 기반으로 빌드하는 것을 추천합니다. nodejs 설치없이도 docker만으로 빌드하고 배포하고 운영하실 수 있습니다.
